Kiket is built to be used by a cross-functional team. The docs are organized around capabilities; this page reverses that and organizes them around **who you are**.

<Cards>
  <Card title="Designer / PM" href="/docs/personas/designer" description="Work on the visual canvas, skip the YAML, get everything shipped." />
  <Card title="Ops engineer" href="/docs/personas/ops" description="YAML + IDE, schema autocomplete, per-state tuning." />
  <Card title="Developer" href="/docs/personas/developer" description="API, SDKs, CLI, MCP, webhooks, CI integrations." />
  <Card title="Compliance officer" href="/docs/personas/compliance" description="Audit trail, evidence export, framework mapping." />
</Cards>

All four personas share the same platform, the same data, and the same audit trail. You can switch between them at any time.
