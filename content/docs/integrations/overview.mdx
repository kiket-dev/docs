Integrations are **evidence-source extensions**. They ingest operational events from the tools your team already uses, normalize them into the event log, attach evidence to cases, and give scanners the context they need to produce explainable findings.

Every extension declares its scopes, Kiket signs callbacks, and admins can revoke access at any time. No mystery OAuth scopes, no hidden mutation path.

## Install one in under a minute

<Steps>
<Step>Open Settings → Evidence Sources.</Step>
<Step>Choose the source that proves your first monitored process, such as GitHub for pull requests or Jira for work item status.</Step>
<Step>Grant the minimum scopes requested by the source.</Step>
<Step>Confirm that source events appear as evidence, scanner input, findings context, and report context.</Step>
</Steps>

## What to verify after install

After connecting an evidence source, use the Process Twin dashboard and Findings Inbox to confirm it is creating value:

- the first-value checklist advances past **Connect an evidence source**
- operational events or evidence records appear for the monitored process
- the scanner can explain at least one finding or confirm no gaps
- each finding shows linked case context, evidence count, and a recommended next action
- generated reports include the source evidence and scanner explanations

If a source connects successfully but no evidence appears, check the source scopes, webhook delivery logs, and ingestion failure visibility before trusting the scanner result.

## How extensions work

1. **Manifest.** Every extension ships an `extension.yaml` declaring scopes, events it receives, and evidence it can emit.
2. **Install = OAuth.** Installing creates an org-level grant; an admin can revoke at any time.
3. **Event ingestion.** The extension receives or pulls source events and submits them to the raw event inbox.
4. **Normalization.** Kiket creates operational events and evidence records linked to workspaces, processes, and cases.
5. **Scanner output.** Findings, remediation actions, reports, and optional integrity proofs reference the same evidence graph.

## Can't find your tool?

Build a custom evidence source: webhook in any language, manifest declaring scopes and event types, one-line install for anyone in your org. See **[Your first extension](/docs/start/first-extension)**.

## Security

- Scopes are least-privilege and visible in Settings.
- Every callback is HMAC-signed; extensions that don't verify get blocked.
- Rate-limited (per extension, per org) to prevent runaways.
- One-click revoke — nothing survives on the extension side.
