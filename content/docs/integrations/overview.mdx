Integrations are **extensions** — signed webhook-based adapters that run outside the core platform. Every extension declares its scopes, Kiket signs every callback, and you see a live activity log of what it does. No mystery OAuth scopes, no shadow automation.

## Install one in under a minute

<Steps>
<Step>From any project, open Settings → Integrations → Marketplace.</Step>
<Step>Click Install on the extension you want. Grant the OAuth scopes it requests.</Step>
<Step>Pick defaults (a channel, a calendar, a repo). You can override per-workflow later.</Step>
<Step>Use the extension from any workflow's automations. Every new action it exposes shows up in the dropdown.</Step>
</Steps>

## How extensions work

1. **Manifest.** Every extension ships an `extension.yaml` declaring scopes, events it wants, actions it exposes.
2. **Install = OAuth.** Installing creates an org-level grant; an admin can revoke at any time.
3. **Webhook calls.** When a workflow triggers the extension, Kiket POSTs a signed payload to the extension's endpoint.
4. **Signed response.** The extension verifies the HMAC, does its work, returns a signed response.
5. **Audit entry.** Both request and response are logged to your audit trail.

## Can't find your tool?

Build a custom extension — webhook in any language, manifest declaring scopes, 1-line install for anyone in your org. See **[Your first extension](/docs/start/first-extension)**.

## Security

- Scopes are least-privilege and visible in Settings.
- Every callback is HMAC-signed; extensions that don't verify get blocked.
- Rate-limited (per extension, per org) to prevent runaways.
- One-click revoke — nothing survives on the extension side.
