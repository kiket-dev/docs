Use this page whenever you need to check if Kiket is experiencing an incident or you need direct help.

## Service status

- Public status page: [status.kiket.dev](https://status.kiket.dev)
- Recommended first step before opening an incident ticket: verify current component status and recent events.

## Support channels

- Product support: [support@kiket.dev](mailto:support@kiket.dev)
- Sales and onboarding: [sales@kiket.dev](mailto:sales@kiket.dev)
- Security disclosure: [security@kiket.dev](mailto:security@kiket.dev)
- Data protection requests (DPA/GDPR): [dpo@kiket.dev](mailto:dpo@kiket.dev)

## Incident reporting checklist

To help us resolve issues quickly, include:

1. Organization and project name.
2. Approximate timestamp and timezone.
3. URL or page where the problem occurred.
4. Screenshot and browser/devices used.
5. Whether [status.kiket.dev](https://status.kiket.dev) showed an active incident.
