You'll create an organization, connect (or create) a **workspace repository**, activate a workflow template, and transition your first issue end-to-end — all in the browser. No credit card, no credentials to configure.

<Callout type="info">
A workspace is a private git repository where your workflow configuration lives as YAML. You never need to know git to use Kiket — the app writes the commits for you. But everything is versioned, diffable, and revertible.
</Callout>

## Walkthrough

<Steps>

<Step>

### Create your account

Visit [app.kiket.dev](https://app.kiket.dev/register) and sign up with email or a Google / Microsoft / GitHub account. You'll land in an empty organization — its name is taken from your email domain, you can rename it later.

</Step>

<Step>

### Pick a setup path

On the **New project** screen, choose one of the three source cards:

- **Copy from Template** — browse the gallery, pick one, click *Use this template*.
- **Use Existing Repo** — connect an existing repo that has a `.kiket/` directory; the app discovers workflows, boards, and issue types automatically.
- **Create New Repo** — we create a new repo for you and scaffold a minimal Kanban workflow.

<Callout title="Why a repo?">
Kiket commits every change (workflow edit, board tweak, approval rule) as a file in your repo. The commit history *is* your audit trail. You don't have to touch git to work — but it's there if you ever need to roll back.
</Callout>

</Step>

<Step>

### Name your project

Enter a project name and 2–10 character key (like `LEGAL` or `ACME`). Invite a teammate if you want company for the first workflow — optional, skippable.

</Step>

<Step>

### Watch the progress screen

The wizard will:

1. Create the workspace repository (if you picked template or fresh).
2. Load the configuration — workflows, boards, issue types, intakes.
3. Set up your default board.
4. Drop you straight into it.

The whole thing takes 10–30 seconds.

</Step>

<Step>

### Open management settings

After your first project is live, use **Settings** as the management hub:

- `/settings/projects` for project CRUD
- `/settings/repositories` for connected repositories
- `/settings/sync` for sync mode/interval and manual sync controls
- `/settings/history` for audit + sync activity

</Step>

<Step>

### Create your first issue

On the board, click **New issue**. Give it a title, pick an issue type (Task, Bug, Feature), and save. Drag it between columns to transition through states. Every transition emits an event; the **Audit** sidebar shows them.

</Step>

<Step>

### See the audit trail

Open the issue detail view and scroll to **Activity**. Every state change, comment, and assignment is logged with actor, timestamp, and hash. When batched (roughly once an hour), these entries are anchored on the Polygon blockchain — at that point they become *externally verifiable*.

<img src="/diagrams/audit-trail.svg" alt="Four steps: action happens, hash computed, anchored on-chain, auditor verifies" />

</Step>

</Steps>
